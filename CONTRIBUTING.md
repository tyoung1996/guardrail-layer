# Contributing to Guardrail Layer

Thank you for considering contributing to Guardrail Layer! We welcome contributions from the community to help improve the project. To make the process smooth and efficient, please follow the guidelines below.

## How to Contribute

### Reporting Issues

If you find a bug or have a feature request, please open an issue on our GitHub repository. Be sure to include:

- A clear and descriptive title.
- A detailed description of the problem or feature.
- Steps to reproduce the issue, if applicable.
- Any relevant screenshots or logs.

### Submitting Pull Requests

We encourage you to submit pull requests (PRs) for bug fixes, enhancements, or new features.

1. **Fork the repository** and create your branch from `main`.
2. **Write clear, concise commit messages.**
3. **Ensure your code follows the existing style and conventions.**
4. **Include tests** for any new functionality or bug fixes.
5. **Run all tests** and ensure they pass.
6. **Submit a pull request** describing your changes and why they are needed.

### Code Style

Please follow the project's coding style to maintain consistency:

- Use 2 spaces for indentation.
- Write clear and descriptive variable and function names.
- Include comments where necessary to explain complex logic.

### Testing

Before submitting a PR, run all tests to verify that your changes do not break existing functionality. Add new tests for any new features or bug fixes.

### Communication

Be responsive to feedback on your pull requests. We may request changes or clarifications before merging.

## Code of Conduct

Please note that this project is released with a [Contributor Code of Conduct](CODE_OF_CONDUCT.md). By participating, you agree to abide by its terms.

## Thank You!

We appreciate your interest in improving Guardrail Layer. Your contributions make a difference!
